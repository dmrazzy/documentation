## Inji Verify: Technical Deep Dive
The main purpose of the demonstration is to provide a technical overview of Inji Verify, a module within the Inji stack that enables verification of verifiable credentials and presentations. The session explains the architecture, components, integration flows, and local setup, helping viewers understand how to use Inji Verify for secure credential verification and presentation sharing. This is valuable for developers and integrators aiming to implement or extend identity verification solutions using Inji.


https://youtu.be/odf_bo38NKI?si=IkE3_6Cz0P3njBrK



1. **Inji Verify Overview**: Inji Verify is a module that supports verification of both verifiable credentials and presentations, implementing the OpenID for Verifiable Presentation protocol.

2. **Core Flows Explained**: The two main flows are VC verification via QR code (upload/scan) and VP sharing using OpenID, each with distinct backend and frontend interactions.

3. **Component Structure**: Inji Verify consists of a backend service (Java Spring), a React-based SDK for integration, and a reference UI, all designed for modular use and easy integration.

4. **Local Development Setup**: Local setup is streamlined using Docker Compose, which orchestrates the backend service, UI, and PostgreSQL database, with clear documentation for configuration.

5. **SDK Integration**: The React SDK exposes components for QR code scanning/upload and OpenID VP sharing, with customizable props and callbacks, making it simple to add verification features to any React application.
