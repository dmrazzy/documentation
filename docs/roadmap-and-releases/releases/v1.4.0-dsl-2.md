# DSL Orchestrator & Packet Creator v1.4.0 — Stories-based Release Notes

## Overview

This release focuses on improved test coverage, reporting, and execution reliability for the MOSIP DSL testing pipeline. 21 new DSL scenarios have been added, covering registration, UIN management, biometric updates, authentication flows, and negative validation cases. Enhancements include better reporting, traceability, notification capture, and execution stability.

## Stories and Delivered Features

### 1. [MOSIP-43461](https://mosip.atlassian.net/browse/MOSIP-43461): Separate Error Report for DSL Test Rig
**Key Features Delivered:**
	- Introduced a dedicated Failed & Skipped HTML report for DSL executions, similar to the API Test Rig. Users can quickly review failing scenarios without downloading the complete execution report.
	- Reports now display pass, fail, and skip percentages along with detailed execution summaries.
	- Enhanced test reporting with execution statistics, including fastest, slowest, and average execution times.

### 2. [MOSIP-44449](https://mosip.atlassian.net/browse/MOSIP-44449): Bug ID Integration in DSL Reports
**Key Features Delivered:**
	- Added Bug ID to Jira linking support in scenario reports for better traceability.
	- Added support to include Bug IDs for known issues directly in DSL reports, with links to Jira tickets for improved traceability.

### 3. [MOSIP-44180](https://mosip.atlassian.net/browse/MOSIP-44180): Notification Capture in DSL Scenarios
**Key Features Delivered:**
	- Added configurable notification handling and request tracking for notification-based flows.
	- Introduced DSL step to capture and report notifications during scenario execution.
	- Implemented a DSL step to capture and log system notifications (UIN creation, VID generation, authentication results) during scenario execution. These are now recorded in DSL/TestNG reports.

### 4. [MOSIP-44268](https://mosip.atlassian.net/browse/MOSIP-44268): Biometric Update Scenario Stabilization
**Key Features Delivered:**
	- Improved execution reliability with enhanced error handling, null-safety checks, and REST error propagation.
	- Implemented selective biometric updates, enabling updates of individual fingerprints or irises.
	- Investigated and resolved failures in Update Biometric scenarios. Fixes ensure stable and reliable execution of biometric update workflows.

### 5. [MOSIP-35453](https://mosip.atlassian.net/browse/MOSIP-35453): Packet Creator Log Traceability
**Key Features Delivered:**
	- Enhanced DSL reporting by improving traceability of Packet Creator interactions. Execution details now better correlate orchestrator runs with Packet Creator activities.
	- Reports now include environment metadata such as commit details, branch links, Docker image information, and runtime environment details.
	- Improved Excel report format with styled layout, colour coding, auto-sizing columns, and frozen headers.
	- Strengthened system security and stability with improved context handling, path validation, and safer file operations.
	- Added a post-execution step to delete all registration centres, ensuring cleaner test environments between executions.
	- Improved test coverage across registration, packet creation, biometric validation, and UIN update workflows.
	- Added 21 new DSL scenarios covering registration, UIN management, biometric updates, authentication flows, and multiple negative validation cases.
