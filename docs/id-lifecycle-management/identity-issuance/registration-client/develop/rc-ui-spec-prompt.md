
# Registration Client UI Spec
- Read through the Registration Client UI Spec attached here
- Also read through Spec from https://docs.mosip.io/1.1.5/modules/registration-client/ui-specification-for-registration-client - this is more human readable format and contains details in tabular format as well as description on usage as to how to use the UI components
- Create a prompt for generating Registration Client UI guide based on the above two documents


## Prompt for Generating Registration Client UI Guide

Please use the following inputs:

1. Learn the entire registration client
2. **Technical Specification**: Review the attached Registration Client UI Spec document - md file
3. If needed you can refer to the registration client repo as well from here https://github.com/mosip/registration-client which is part of mosip github org


### Requirements

Generate a user guide that includes:

- **UI Component Catalog**: Document all UI components with descriptions
- **Usage Instructions**: Step-by-step guidance for each UI element
- **Tabular Reference**: Convert technical specs into easy-to-read tables
- **Navigation Flow**: Explain the user journey through different screens
- **Field Specifications**: Detail input fields, validations, and data types
- **Best Practices**: Include usage recommendations and common scenarios
- **Accessibility Features**: Document keyboard shortcuts and accessibility options

### Output Format

Structure the guide with:
- Clear section headings
- Visual hierarchy using markdown
- Tables for component specifications
- Code examples where applicable
- Cross-references between related sections
