# Glossary

Here’s your glossary formatted as a table:

<table><thead><tr><th width="187">Term</th><th>Description</th></tr></thead><tbody><tr><td>MOSIP</td><td>Modular open-source identity platform.</td></tr><tr><td>FTM</td><td>Foundational Trust Provider.</td></tr><tr><td>Open Source</td><td>Open source refers to freely available software for anyone to use, modify, and share, ensuring trusted and transparent technology that adheres to global standards.</td></tr><tr><td>Vendor-Neutral</td><td>Offering the ability to integrate with a wide range of compliant technologies.</td></tr><tr><td>Pre-registration</td><td>A resident-facing web-based portal that allows a resident to provide registration data, upload document proofs, and book an appointment with a registration centre to complete the rest of the registration process.</td></tr><tr><td>Registration</td><td>A process that allows a resident to provide demographic information and biometrics by visiting a registration centre.</td></tr><tr><td>ID Schema</td><td>A standard JSON schema that defines a dataset that can be stored in the Identity repository for a resident.</td></tr><tr><td>UIN</td><td>A unique number assigned to a resident. UIN never changes and is non-revocable. UIN is randomized, so one should not be able to derive any Personal Identifiable Information (PII) from the number itself.</td></tr><tr><td>VID</td><td>Virtual ID is an alias identifier that can be used for authentication transactions.</td></tr><tr><td>Handle</td><td>A distinctive identifier created/provided by the resident during the registration.</td></tr><tr><td>MDS</td><td>MOSIP Device Service.</td></tr><tr><td>MV</td><td>Manual Verification.</td></tr><tr><td>MA</td><td>Manual Adjudication.</td></tr><tr><td>ABIS</td><td>Automated Biometric Identification System.</td></tr><tr><td>SDK</td><td>Software Development Kit.</td></tr><tr><td>SBI</td><td>Secure Biometric Device Interface.</td></tr><tr><td>CBEFF</td><td>Common Biometric Exchange File Format.</td></tr><tr><td>CTK</td><td>Compliance Tool Kit.</td></tr><tr><td>AID</td><td>Application ID.</td></tr><tr><td>Device Provider</td><td>An entity that manufactures or imports the devices in their name. This entity should have legal rights to obtain an organization-level digital certificate from the respective authority in the country.</td></tr><tr><td>FTM Provider</td><td>An entity that manufactures or guarantees the trustworthiness of the foundational trust module. This can be the device provider as well.</td></tr><tr><td>Device</td><td>Hardware capable of capturing biometric information.</td></tr><tr><td>L1 Certified Device/L1 Device</td><td>A device certified as capable of performing encryption in line with this specification in its trusted zone.</td></tr><tr><td>L0 Certified Device/L0 Device</td><td>A device certified as one where the encryption is done on the host machine device driver or the SBI service.</td></tr><tr><td>FTM Provider Certificate</td><td>A digital certificate issued to the "Foundational Trust Provider". This certificate proves that the provider has successfully gone through the required Foundational Trust Provider evaluation. The entity is expected to keep this certificate in secure possession in an HSM. All the individual FTM trust certificates are issued using this certificate as the root. This certificate would be issued by the countries in conjunction with a trusted registry.</td></tr><tr><td>Device Provider Certificate</td><td>A digital certificate issued to the "Device Provider". This certificate proves that the provider has been certified for L0/L1 respective compliance. The entity is expected to keep this certificate in secure possession in an HSM. All the individual trust certificates are issued using this certificate as the root. This certificate is issued by the countries in conjunction with a trusted registry.</td></tr><tr><td>Registration</td><td>The process of applying for a Foundational ID.</td></tr><tr><td>KYC</td><td>Know Your Customer. The process of providing consent to perform profile verification and update.</td></tr><tr><td>Auth</td><td>The process of verifying one’s identity.</td></tr><tr><td>FPS</td><td>Frames Per Second.</td></tr><tr><td>Management Server</td><td>A server run by the device provider to manage the life cycle of the biometric devices.</td></tr><tr><td>Device Registration</td><td>The process of a device obtaining a certificate from the management server.</td></tr><tr><td>Signature</td><td>All signatures should be as per RFC 7515.</td></tr><tr><td>Header in signature</td><td>The header in the signature means the attribute with "alg" set to RS256 and x5c set to base64encoded certificate.</td></tr><tr><td>Payload</td><td>Payload is the byte array of the actual data, always represented as base64urlencoded.</td></tr><tr><td>Signature</td><td>Base64 url encoded signature bytes.</td></tr><tr><td>ISO format timestamp</td><td>ISO 8601 with format yyyy-mm-ddTHH:MM:ssZ (Example: 2020-12-08T09:39:37Z). This value should be in UTC (Coordinated Universal Time).</td></tr></tbody></table>
